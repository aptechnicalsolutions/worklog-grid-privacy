# WorkLog Grid for Jira – Privacy Policy

**Effective Date:** [Insert Date]

AP Technical Solutions LLC (“we”, “our”, or “us”) provides the WorkLog Grid for Jira application (the “App”). This Privacy Policy explains how the App processes data.

## 1. Overview

WorkLog Grid for Jira is built using Atlassian Forge and runs entirely on Atlassian’s cloud infrastructure. The App does not operate its own servers and does not transmit data to external systems outside Atlassian. The App processes customer data only as necessary to provide its functionality within the Atlassian environment and does not independently use customer data for its own purposes.

## 2. Data Accessed

The App accesses the following data within Jira solely to provide its functionality:

- **Issue metadata** (issue keys, summaries)
- **Worklog data** (time logged, dates, author)
- **User data** (display names, account identifiers, and avatar URLs when you use the “Other” user feature)
- **Project metadata** (project keys and names)

This data is accessed using Atlassian’s secure APIs and only within the permissions granted during installation.

## 3. Data Storage

The App does not store personal data outside of Atlassian infrastructure.

**Forge storage (Atlassian-hosted):** The App uses Atlassian Forge key-value storage for:

- **User preferences** (e.g., date range, project filter, “Me” vs “Other” user choice). When you choose to view another user’s timesheet (“Other”), the App stores the selected user’s Atlassian account identifier and display name solely to restore your previously selected view. No additional profiling or processing is performed. This data is stored securely within Forge and is not accessible outside the App.
- **Short-lived cache** (e.g., 90 seconds) to improve performance when refreshing the grid. Cached data is scoped to the requesting user and request parameters and expires automatically.

The App does not maintain external databases or third-party storage systems.

## 4. Data Sharing

The App does not sell, share, or transfer customer data to third parties. All processing occurs within Atlassian’s cloud environment.

## 5. Data Retention

- **Cache:** Grid results are cached for a short period (e.g., 90 seconds) and then expire; no long-term retention of cached data.
- **Preferences:** Stored until you change them or the App is uninstalled, solely to restore your settings (date range, project, selected user for “Other” view).

The App does not retain personal data beyond what is necessary for this functionality.

## 6. Security

WorkLog Grid for Jira relies on Atlassian’s Forge platform security model. All authentication, authorization, and infrastructure security are managed by Atlassian.
